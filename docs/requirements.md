# 系统需求规格说明

## 0. 当前开发要求

本项目目前只做三个端都能用的三个功能：个人信息管理、教师树洞、思政培训活动。三个端分别为教师端、二级党委管理员端、校级管理员端。其他功能目前不要，不能作为当前版本的开发范围、菜单入口、接口范围或验收内容。

## 1. 用户角色

| 角色 | 使用端 | 当前权限范围 |
| --- | --- | --- |
| 教师 | 微信小程序教师端 | 查看和维护本人基础信息，提交和跟踪教师树洞诉求，查看并参与思政培训活动。 |
| 二级党委管理员 | 管理后台二级党委端 | 查看本端个人信息，查看本院教师树洞事项，查看本院思政培训活动。 |
| 校级管理员 | 管理后台校级端 | 查看本端个人信息，查看全校教师树洞事项，查看全校思政培训活动。 |

## 2. 功能性需求

### 2.1 三端共同保留功能

| 功能 | 需求描述 |
| --- | --- |
| 个人信息管理 | 支持查看姓名、工号或账号、所属单位、手机号、邮箱等基础信息；后续仅允许维护非敏感联系方式。 |
| 教师树洞 | 支持教师端提交诉求；三个端均可查看与本端范围相关的诉求列表、办理进度与处理结果；教师端支持满意度评价。 |
| 思政培训活动 | 支持查看培训列表；教师端支持报名参与、提交学习记录或学习成果、查看个人学习台账；管理端保留活动查看能力。 |

### 2.2 暂不开发范围

| 范围 | 说明 |
| --- | --- |
| 其他业务 | 思想状况调研、师德建设季报、学子说、数据统计与导出、系统运维暂不开发。 |
| 其他管理功能 | 角色权限配置、通用支撑模块、管理员审批、管理员分派办理、报表生成暂不开发。 |
| 其他页面入口 | 除个人信息管理、教师树洞、思政培训活动外，不新增其他菜单、Tab 或页面入口。 |

## 3. 非功能性需求

| 指标 | 目标 |
| --- | --- |
| 小程序页面加载时间 | 不超过 1.5 秒。 |
| 管理后台页面加载时间 | 不超过 2 秒。 |
| 简单查询接口响应时间 | 不超过 500ms。 |
| 身份认证 | 开发阶段使用占位 token，正式环境接入学校统一身份认证。 |
| 数据安全 | 树洞内容等敏感数据需要按端和范围进行隔离。 |
| 传输安全 | 正式环境数据传输使用 HTTPS。 |
| 易用性 | 三个端的三项核心流程清晰可用，页面风格简洁统一。 |
| 兼容性 | 小程序支持微信 8.0 及以上版本；后台兼容主流浏览器。 |

## 4. 当前项目落地状态

| 类型 | 当前状态 |
| --- | --- |
| 微信小程序教师端 | 已保留首页、个人信息管理、教师树洞、思政培训页面骨架。 |
| 管理后台 | 已保留概览、个人信息管理、教师树洞、思政培训页面骨架，作为二级党委管理员端和校级管理员端的当前承载。 |
| 后端 API | 已保留认证、个人信息、教师树洞、思政培训接口骨架。 |
| 数据库 | 已收窄为用户、树洞、培训、培训记录和附件相关表。 |
